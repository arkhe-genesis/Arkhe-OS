## 🧠 1. Currículo Canônico de IA — Engenheiro, Builder, Líder

O currículo é organizado em três trilhas paralelas que convergem no domínio comum do ecossistema Arkhe. Cada módulo tem carga horária estimada e projeto final prático integrado aos substratos existentes.

```

║           ARKHE AI CURRICULUM — THREE PATHS, ONE CATHEDRAL      ║
╠══════════════════════════════════════════════════════════════════╣
║                                                                  ║
║  🛠️ ENGINEER (Build Smarter)                                    ║
║  ├─ Sem 1: AI-Assisted Code Generation (Copilot, Cursor)        ║
║  ├─ Sem 2: Test & Verification Automation (ConRAG, ZK-proofs)   ║
║  ├─ Sem 3: Refactoring with AI (Refactor Engine 6102)           ║
║  └─ Sem 4: AI-Driven Code Review & Documentation (Docs Gen)     ║
║                                                                  ║
║  🚀 BUILDER (Ship Faster)                                       ║
║  ├─ Sem 1: Prompt Engineering for Rapid Prototyping             ║
║  ├─ Sem 2: Automated CI/CD with AI (CI Triage 6133)             ║
║  ├─ Sem 3: AI-Optimized Build Pipelines (Cache 6105, Perf 6142) ║
║  └─ Sem 4: Multi-Platform Deployment with AI (Platform 6143)    ║
║                                                                  ║
║  👑 LEADER (Decide Better)                                       ║
║  ├─ Sem 1: AI-Enhanced Strategic Analysis (Scenario Modeling)   ║

║  ├─ Sem 3: AI-Powered Forecasting & Metrics (eBPF Dashboard)    ║
║  └─ Sem 4: AI Governance & Compliance Automation (Golden Dome)   ║
║                                                                  ║
║  🔗 CAPSTONE PROJECT (All Paths):                                ║
║  "Build and Govern an AI-Verified Arkhe Package Ecosystem"       ║
║  Uses: arkp CLI + ConRAG + Mythos Gate + QIP + TemporalChain     ║
╚══════════════════════════════════════════════════════════════════╝
```

Cada semestre corresponde a um módulo prático com código e métricas. Exemplo do **Sem 1 Engineer**:

- **Objetivo:** Usar GitHub Copilot para gerar uma API REST em Go (Substrato 6200), verificar a segurança com o Mythos Gate, e publicar o pacote no registry.
- **Métrica:** Número de sugestões aceitas × índice de vulnerabilidades detectadas.

## 🗺️ 2. Mapa de Ferramentas Reais de IA × Habilidades Arkhe

| Habilidade | Ferramenta de IA | Uso no Ecossistema Arkhe | Exemplo Canônico |
|------------|------------------|---------------------------|------------------|
| **Geração de código** | **GitHub Copilot** | Acelera escrita de novos plugins, APIs em Go (Substrato 6200) | `arkp new go-api` + Copilot sugere rotas Gin |
| **Refatoração inteligente** | **Cursor** (editor AI‑first) | Refatora código Python do `ConRAG` ou `MythosGate` com consciência do UAST | "Cursor, extract method for `detect_exit_trigger()`" |
| **Revisão de código** | **Claude 3.5 Sonnet** (via chat) | Revisa `ArtBlocks` antes da publicação, aponta vulnerabilidades éticas | `claude.ai: "Revise este pacote e compare com as regras do Mythos Gate"` |
| **Testes automatizados** | **Copilot + Pytest** | Gera casos de teste para plugins de auditoria ética | `# Copilot: "generate tests for license_audit_plugin"` |
| **CI/CD otimização** | **GitHub Actions + Claude** | Sugere melhorias no pipeline de build e cache | `arkp ci triage --suggest` (com IA) |
| **Análise de dependências** | **Dependabot + IA** | Atualiza dependências com verificação de compatibilidade semântica (SemVer) | `arkp deps upgrade --ai-suggest` |
| **Dashboard & Métricas** | **Claude + Prometheus** | Interpreta métricas do eBPF e sugere otimizações de saúde do ecossistema | `"Analise este dashboard e identifique o pacote com maior risco"` |
| **Decisão ética** | **Mythos Gate + Claude** | Avalia decisões de publicação com explicação humana | `arkp publish --orcid ... --explain` (usa IA para gerar rationale) |
| **Tradução multi-idioma** | **GPT‑4o / Claude** | Traduz automaticamente strings da UI para i18n | `arkp i18n translate --from en --to pt-BR` |
| **Documentação** | **Claude + Copilot** | Gera documentação a partir de docstrings e código | `arkp doc --ai-generate` |

Cada linha da tabela é um **módulo prático** do currículo, onde o aprendiz utiliza a ferramenta real em um cenário Arkhe.
